Frostbite

A frontend for Dragonrealms MUD

3 Scripting

The Frostbite scripting support is provided by the Ruby scripting engine and will therefore require the Ruby runtime libraries to be installed.


Ruby scripting engine adds a flexible and powerful scripting support for the game client. It is easy to use for beginners and yet it still enables more complex scripting possibilities for advanced users.


High level description

Most of the scripting functionality is created in Ruby and it can be easily re-adjusted to every specific need. The game client provides game text over standard output, while rest of the utility functions are available as part of a dynamic library file "data.dll/so", which can be accessed by any other process -- including the Ruby scripting module.

The Ruby side API is placed in the "scripts/lib" folder and should only be modified at your own risk. The game client always tries to execute the "main.rb" script and it specifies the script file as part of the passed arguments that will remain invisible to the user. The specified file will be in turn executed by the "main.rb" script.

For example: game client -> main.rb "braid.rb vines" -> braid.rb

The scripting API includes:

  • main.rb - general functions used by scripts;
  • models.rb - wraps and provides the game data that can be used in scripts;
  • data.rb - interface for game data library;
  • ruby_goto.rb - enables to use Wizard style labels in script logic;
  • observer.rb - observer api, allows to react to text events in continuous game flow.

The scripting API functions are described in more detail in the API section.
